This one is built as a system of small, independently deployable surfaces, so each concern evolves on its own cadence without coupling. The main site stays small and stable; everything that changes frequently lives behind a subdomain.\r\n\r\nThat separation is the point. It mirrors how production platforms are actually run, and it keeps every surface cheap to reason about, deploy, and roll back.\r\n\r\n---\r\n\r\n## Ecosystem\r\n\r\nThis site is the hub. static site to Pages on `main`. Pull requests get their own preview deployment (scoped via `VITE_BASE_PATH`), and a `404.html` fallback lets React Router own deep links. Concurrency cancels superseded runs.\r\n\r\n**`ci.yml` — Build & Push.** Builds the `Dockerfile` for `linux/amd64` and `linux/arm64` (Buildx + QEMU), pushes by digest to GHCR and Docker Hub, then merges them into one multi-arch manifest. Tagged `latest`, `sha-<short>`, build date, and semver; PRs build without pushing.\r\n\r\nProduction traffic is served by the **static Pages build**. The container image is an independent, portable deliverable — not in the live request path — that runs anywhere nginx can. Its bundled `nginx.conf` is hardened for a static SPA: security headers (CSP, HSTS, `X-Frame-Options`), immutable asset caching with an uncached `index.html`, gzip, and a `try_files` fallback to `index.html` for client-side routing.\r\n\r\n### Published images\r\n\r\nThe container pipeline publishes a **multi-arch image** (`linux/amd64`, `linux/arm64`) to two registries:\r\n\r\n| Registry | Image |\r\n| --- | --- |\r\n| **GitHub Container Registry** | `ghcr.io/ibtisam-iq/ibtisam-iq` |\r\n| **Docker Hub** | `docker.io/mibtisam/mibtisam` |\r\n\r\nTags include `latest`, `sha-<short>`, the build date, PR refs, and semver. Pull and run either registry:\r\n\r\n```bash\r\n# GHCR\r\ndocker run --rm -p 8080:8080 ghcr.io/ibtisam-iq/ibtisam-iq:latest\r\n\r\n# Docker Hub\r\ndocker run --rm -p 8080:8080 mibtisam/mibtisam:latest\r\n\r\n# → http://localhost:8080\r\n```\r\n\r\n---\r\n\r\n## Architecture at a glance\r\n\r\n```\r\n                          ┌──────────────────────┐\r\n                          │      Cloudflare      │\r\n                          │          DNS         │\r\n                          └───────────┬──────────┘\r\n                                      │\r\n        ┌──────────────────┬────────┬─────────────────────┬──────────────────┐\r\n        │                  │                      │                  │\r\n  ibtisam-iq.com    projects.ibtisam-iq    runbook.ibtisam-iq   blog / nectar / …\r\n   (this repo)        (separate repo)        (separate repo)      (separate repos)\r\n        │\r\n   push to main\r\n        │\r\n   ┌────┴─────────────────────────────────────────────────────┐\r\n   │  GitHub Actions                                                │\r\n   │  ├─ pages.yml  → lint → build → deploy to Pages (+ PR preview) │  ← live\r\n   │  └─ ci.yml     → multi-arch build → push to GHCR + Docker Hub  │  ← images\r\n   └──────────────────────────────────────────────────────────────────── ┘\r\n```\r\n\r\nProduction is **static, served from GitHub Pages**, with DNS handled by Cloudflare. No SSR by design — the trade is predictable, fast, and operationally near-zero.\r\n\r\n---\r\n\r\n## Project structure\r\n\r\n```text\r\nportfolio-site/\r\n├── .github/workflows/      # pages.yml (Pages + previews) · ci.yml (multi-arch images)\r\n├── public/                 # Static assets (favicon, cv.pdf, etc.)\r\n├── src/\r\n│   ├── components/         # Reusable UI (Navbar, etc.)\r\n│   ├── App.tsx             # Root app component\r\n│   ├── main.tsx            # Entry point\r\n│   └── index.css           # Global styles / Tailwind layer\r\n├── Dockerfile              # Multi-stage build → rootless nginx:alpine\r\n├── nginx.conf              # Hardened nginx config (port 8080, CSP, caching)\r\n├── index.html              # HTML entry — SEO, meta, favicon\r\n├── eslint.config.js\r\n├── postcss.config.js\r\n├── tailwind.config.js\r\n├── tsconfig.json           # + tsconfig.app.json, tsconfig.node.json\r\n├── vite.config.ts\r\n└── package.json\r\n```\r\n\r\nThe layout is intentionally **flat and explicit** — structure that fits in a single mental model.\r\n\r\n---\r\n\r\n## Local development\r\n\r\n**Prerequisites:** Node.js and npm. CI runs on Node 24; the container build pins Node 22 (`22.14-alpine`). Any recent LTS works locally.\r\n\r\n```bash\r\n# Install exact, reproducible dependencies from the lockfile\r\nnpm ci\r\n\r\n# Start the Vite dev server\r\nnpm run dev\r\n\r\n# Type-check and build for production (tsc -b && vite build)\r\nnpm run build\r\n\r\n# Lint the codebase\r\nnpm run lint\r\n\r\n# Preview the production build locally\r\nnpm run preview\r\n```\r\n\r\n### Run the container\r\n\r\nThe image is a **3-stage build** (`deps` → `builder` → `production`) that ends on `nginx:alpine` with no Node toolchain in the final layer. It runs **rootless** (`USER nginx`), exposes **8080**, ships a `HEALTHCHECK`, and uses `STOPSIGNAL SIGQUIT` for graceful connection draining.\r\n\r\nPull a published image, or build locally for production parity:\r\n\r\n```bash\r\n# Run the published image (GHCR)\r\ndocker run --rm -p 8080:8080 ghcr.io/ibtisam-iq/ibtisam-iq:latest\r\n```\r\n\r\n```bash\r\n# Build from source (CI injects BUILD_DATE / GIT_SHA as OCI labels)\r\ndocker build -t portfolio-site .\r\ndocker run --rm -p 8080:8080 portfolio-site\r\n\r\n# → http://localhost:8080\r\n```\r\n\r\n---\r\n\r\n## Local CI Testing (`act`)\r\n\r\nBoth pipelines can be run locally using [`act`](https://github.com/nektos/act).\r\n\r\n```bash\r\n# Run the Pages build pipeline (lint → build → CNAME → 404)\r\nact push \\\r\n  -W .github/workflows/pages.yml\r\n\r\n# Run the Docker build pipeline (QEMU → Buildx → build)\r\nact push \\\r\n  -W .github/workflows/ci.yml\r\n```\r\n\r\n> [!NOTE]\r\n> Registry logins, metadata extraction, image pushes, artifact uploads, and GitHub Pages deployment are automatically skipped via `if: ${{ !env.ACT }}` guards — `act` sets the `ACT` environment variable automatically.\r\n> For `ci.yml`, the build is pinned to `linux/amd64` locally (required for `load: true`) and the image is loaded into your local Docker daemon as `mibtisam/mibtisam:local`.\r\n\r\n---\r\n\r\n## Deployment model\r\n\r\n- **Production is static.** Served from GitHub Pages; no application server in the request path.\r\n- **CI builds and deploys.** Pushes to `main` trigger the GitHub Actions workflows — no manual deploys.\r\n- **Two artifacts.** The same source produces both the static Pages deploy and a container image published to GHCR and Docker Hub.\r\n- **PR previews.** Pull requests deploy to isolated preview paths before anything reaches `main`.\r\n- **DNS via Cloudflare.** Domain and subdomain routing are handled at Cloudflare; the custom domain is bound through a `CNAME` in the build.\r\n- **No SSR, by design.** The content doesn't need it; avoiding it removes a class of runtime failure.\r\n\r\n---\r\n\r\n## Status\r\n\r\n**Actively evolving.** New systems, documentation, and projects land incrementally as they reach a production-ready bar.
